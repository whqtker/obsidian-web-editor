# UI/UX 전문가 리뷰

전체 코드베이스(`src/components/`, `src/index.css`)를 대상으로 UI 품질을 분석하고 감각적이고 세련된 디자인으로 발전시킨다.

## 리뷰 프로세스

### 1단계: 전체 UI 스캔 및 이슈 식별

`src/components/` 전체와 `src/index.css`를 스캔하여 아래 기준에 따른 이슈를 찾아라:

**시각적 일관성**
- 색상 토큰 불일치 (디자인 시스템 외의 색상 사용)
- 간격/패딩 불균형 (동일 역할에 다른 간격)
- 타이포그래피 불일치 (font-size, font-weight 혼용)
- 테두리/라운딩 불일치

**인터랙션 품질**
- hover 상태 누락 또는 불일치
- focus 링 누락 (키보드 접근성)
- active/pressed 상태 부재
- transition 누락 (상태 변화 시 딱딱한 전환)
- 로딩 상태의 시각적 피드백 부족

**레이아웃 & 공간 활용**
- 불필요한 공간 낭비
- 정렬 불일치
- overflow 처리 미흡 (긴 파일명, 긴 경로)
- 빈 상태(empty state) 디자인 부재 또는 미흡

**마이크로 인터랙션**
- 의미 있는 애니메이션 부재 (목록 진입, 패널 전환)
- 스켈레톤 로딩 미적용
- 성공/실패 피드백의 시각적 명확성

**접근성**
- 키보드 내비게이션 미지원
- 포커스 트래핑 부재 (모달)
- 색상 대비 부족 (WCAG AA 기준)
- aria 속성 누락

### 2단계: 이슈 리포트

발견된 이슈를 아래 형식으로 카테고리별 정리하여 사용자에게 제시:

```
## UI 리뷰 리포트

### 🎨 시각적 일관성
- [ ] 파일:라인 — 설명 + 개선 제안

### ✨ 인터랙션 품질
- [ ] 파일:라인 — 설명 + 개선 제안

### 📐 레이아웃
- [ ] 파일:라인 — 설명 + 개선 제안

### 💫 마이크로 인터랙션
- [ ] 파일:라인 — 설명 + 개선 제안

### ♿ 접근성
- [ ] 파일:라인 — 설명 + 개선 제안
```

각 이슈에 구체적인 Tailwind 클래스 변경 제안을 포함하라. 사용자의 승인을 받은 후 다음 단계로 진행한다.

### 3단계: 수정 적용

승인된 이슈를 순차적으로 수정한다. 각 수정마다:
1. Before/After의 시각적 차이를 설명
2. Tailwind 클래스 변경 적용
3. 필요 시 `src/index.css`에 커스텀 CSS 추가

### 4단계: 검증

모든 수정 완료 후:
1. `npm run build` 실행 — 빌드 성공 확인
2. `npm run lint` 실행 — lint 에러 없음 확인
3. 변경된 컴포넌트 목록과 시각적 변화 요약 제시

## 디자인 시스템

### 색상 토큰
- 배경 계층: bg-gray-950(배경) → bg-gray-900(셸) → bg-gray-800(카드/패널) → bg-gray-700(입력)
- 텍스트 계층: text-white(제목) → text-gray-300(본문) → text-gray-400(라벨) → text-gray-500(힌트)
- 강조색: blue-600(CTA), yellow-500(dirty), red-600(삭제), green-800(성공)

### 간격
- px-3 py-1.5(소), px-4 py-2(중), p-4(카드), gap-2~3

### 인터랙션 패턴
- 모든 클릭 가능 요소에 transition-colors 필수
- hover 상태 명확히 표시 (bg 변경 또는 text 밝기 증가)
- 모달: fixed inset-0 bg-black/50 + 중앙 카드
- 그룹 호버 패턴: group + hidden group-hover:block
- 포커스 링: focus:outline-none focus:ring-1 focus:ring-blue-500
- 키보드 내비게이션: 모달 Escape 닫기, 목록 Arrow 이동
- 버튼 disabled: bg-gray-700 text-gray-500

## 디자인 원칙

1. **일관성** — 같은 역할의 요소는 같은 스타일. 디자인 토큰 준수
2. **계층** — 시각적 중요도에 따른 명확한 계층 (크기, 색상, 굵기)
3. **여백** — 충분한 breathing room. 답답하지 않은 레이아웃
4. **피드백** — 모든 사용자 액션에 즉각적인 시각적 응답
5. **절제** — 과도한 장식 지양. 기능에 봉사하는 미학

## 주의사항

- 기능 변경 금지 — 시각적/인터랙션 개선만 수행
- Tailwind 유틸리티 우선 — 커스텀 CSS는 최소한으로
- 외부 라이브러리 추가 시 반드시 사용자 승인 필요
- CodeMirror 에디터 내부 스타일은 oneDark 테마에 위임 (직접 수정 지양)
- 다크 테마 전용 — 라이트 모드 고려 불필요
